# ArgDigest 0.8.0 Release Notes (Draft)

## Release Focus

`0.8.0` is the release-candidate milestone on the path to `1.0.0`.
This release consolidates API freeze work and strengthens packaging/runtime validation.

## Highlights

- Public API contract locked and covered by dedicated tests.
- API reference aligned with exported public surface (`argdigest.__all__`).
- CI and docs workflows now validate wheel build/install/import explicitly.
- Packaging configuration updated to include all `argdigest*` subpackages in wheels.
- Coverage and test suite strengthened for decorator, registry, loader, and integration branches.
- Integration confidence checks executed against representative MolSysMT usage paths.

## Quality and Validation

- CI workflow: green.
- Sphinx docs workflow: green.
- Wheel build/install/import regression checks: green.
- Conda package build/upload path validated at `0.7.0` tag.

## Compatibility Notes

- No planned breaking API changes remain before `1.0.0`.
- No deprecations are introduced in this RC milestone.

## Known Notes

- One legacy compatibility warning remains expected in tests:
  `DigestNotDigestedWarning` for `syntax` in legacy compatibility coverage.
  This is currently treated as intentional baseline behavior.

## Next

- Continue with final stabilization tasks toward `1.0.0`.
