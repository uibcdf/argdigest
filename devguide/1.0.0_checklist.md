# 1.0.0 Stability and Ecosystem Checklist

This checklist merges ArgDigest's internal release path with ecosystem-level
requirements observed in `../molsyssuite/devguide`.
It is the operational gate for shipping `1.0.0` without ambiguity.

## Status Legend

- `[x]` complete and verified
- `[ ]` pending
- `[-]` intentionally deferred to post-1.0

## A) Core 1.0 must-haves (release blocking)

- [ ] freeze and verify precedence for configuration discovery (`runtime > env > _argdigest.py`).
- [ ] keep `arg_digest` behavior stable across package, registry, decorator, and mixed styles.
- [ ] guarantee deterministic digester discovery and execution order.
- [ ] complete API contract lock for public decorators, errors, and pipelines.
- [ ] verify docs/API/runtime consistency for all exported public symbols.

## B) Cross-library interoperability must-haves (release blocking)

- [ ] document and enforce compatibility matrix with sibling libraries (`smonitor`, `depdigest`, `pyunitwizard`).
- [ ] ensure all critical public API paths emit coherent SMonitor signals.
- [ ] ensure ArgDigest diagnostics include stable `CODE` and machine-readable `hint`.
- [ ] verify optional dependency behavior never hard-requires heavy stacks without DepDigest-compatible flow.

## C) Validation semantics and UX must-haves (release blocking)

- [ ] ensure contract failures are friendly and concise in `user` profile.
- [ ] ensure `dev` profile exposes full digestion trace with actionable context.
- [ ] map `pyunitwizard` conversion failures into ArgDigest contract errors with caller context.
- [ ] validate strictness modes (`raise`, `warn`, `silent`) against documented behavior.
- [ ] verify warning/error taxonomy is consistent across registry and package styles.

## D) Performance and reliability must-haves (release blocking)

- [ ] validate `inspect.signature` caching strategy under repeated high-frequency calls.
- [ ] verify no regression in decorator overhead versus current baseline.
- [ ] verify import-time behavior remains lightweight and deterministic.
- [ ] ensure thread-safe/read-safe behavior for caches and registries in common usage patterns.

## E) Tooling and release gates must-haves (release blocking)

- [ ] CI workflow green (tests, lint, coverage threshold).
- [ ] docs workflow green and `gh-pages` publication successful.
- [ ] wheel install/import check fails fast if package import is broken.
- [ ] conda build/upload workflow validated for current release tag.
- [ ] release notes and migration notes finalized for `1.0.0`.
- [ ] validate ecosystem health check flow (SMonitor/ArgDigest/DepDigest/PyUnitWizard) in an integration scenario.

## F) Cross-layer end-to-end criterion (release blocking)

- [ ] validate that a `pyunitwizard` unit failure is re-emitted by ArgDigest as a contract error with caller context.
- [ ] validate that the emitted ArgDigest diagnostic includes stable `CODE` and machine-readable `hint`.
- [ ] validate dependency remediation hint path (`depdigest`) when failure cause is missing optional stack.
- [ ] confirm this cross-layer behavior is coherent in `user` and `dev` profiles.

## G) Strong should-haves (target for 1.0 if feasible)

- [ ] add explicit integration confidence tests against MolSysMT representative paths.
- [ ] provide at least one end-to-end example per digestion style in docs/showcase.
- [ ] add negative-path tests for malformed `_argdigest.py` and invalid registry wiring.
- [ ] provide an explicit troubleshooting page for end users receiving ArgDigest/SMonitor messages.

## H) Deferred research and future vision (non-blocking for 1.0)

- [-] adaptive validation throttling (trust-based runtime validation reduction).
- [-] autonomous or AI-assisted semantic coercion for ambiguous argument inputs.
- [-] autonomous fuzzing generation from digestion contracts.
- [-] dynamic/JIT dependency provisioning during runtime.
- [-] counterfactual triage and self-healing workflows driven by runtime diagnostics.

## Final release criterion

ArgDigest is ready for `1.0.0` when sections A-F are fully checked and no
open blocker remains in digestion core, configuration resolution, diagnostics,
or ecosystem compatibility.
