# 1.0.0 Release Notes and Migration Summary (Draft)

This draft is the release-ready narrative for promoting ArgDigest from `0.9.x`
to `1.0.0` after consolidation closure.

## Release intent

ArgDigest `1.0.0` declares a stable contract for:
- digestion style behavior (`package`, `registry`, `decorator`, `auto`),
- configuration precedence (`runtime > env > _argdigest.py`),
- catalog-backed diagnostics (`CODE` + `hint`) with SMonitor integration,
- optional dependency routing through DepDigest-compatible flows.

## What is stable in 1.0.0

1. Public API surface
- `arg_digest`
- `arg_digest.map`
- `argument_digest`
- `register_pipeline`
- `get_pipelines`

2. Digestion/runtime behavior
- deterministic digester discovery order,
- strictness aliases and semantics (`raise/error`, `warn/warning`, `silent/ignore/none`),
- standardizer hook contract (`standardizer(caller, kwargs)`).

3. Diagnostics contract
- catalog-backed warnings/exceptions with stable `code` and `hint`,
- user/dev profile readability parity,
- cross-layer error propagation with PyUnitWizard and DepDigest remediation hints.

## Migration summary from pre-1.0 lines

From `0.6.x`/`0.7.x` to `1.0.0`:
- keep existing decorator usage; no breaking decorator rename is required,
- prefer explicit `_argdigest.py` defaults for maintainability,
- move legacy inline validation into digesters/pipelines incrementally,
- use `auto` only as migration stage, not as permanent endpoint.

From `0.8.x`/`0.9.x` RC lines to `1.0.0`:
- no intended public API break,
- keep current CI/docs/conda release gates as mandatory,
- keep shared collective E2E path green across sibling libraries.

## Integrator checklist (host library)

1. Configuration
- provide `<host_package>/_argdigest.py` in package root,
- define `DIGESTION_SOURCE`, `DIGESTION_STYLE`, `STRICTNESS`.

2. Digester architecture
- choose one primary style (`package` or `registry`) for long-term maintenance,
- keep `decorator` style for local/module-specific flows.

3. Diagnostics
- keep SMonitor configured and catalog-aligned,
- verify errors shown to end users contain actionable hints.

4. Optional dependencies
- declare and route optional stacks via DepDigest-compatible patterns.

## Known non-goals for 1.0.0

- adaptive validation throttling,
- AI-assisted semantic coercion,
- runtime JIT dependency provisioning.

These remain post-1.0 topics and are intentionally out of scope for stable release.
