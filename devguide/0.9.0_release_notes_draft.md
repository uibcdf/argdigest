# ArgDigest 0.9.0 Release Notes (Draft)

## Release Position

`0.9.0` is the final release-candidate milestone before `1.0.0`.
It intentionally contains the complete target feature set for `1.0.0` and is
followed by a short consolidation window under real usage.

## What 0.9.0 consolidates

- Full 1.0 contract coverage across:
  - configuration precedence (`runtime > env > _argdigest.py`),
  - multi-style digestion behavior (package/registry/decorator/auto),
  - diagnostics and cross-layer error mapping.
- Compatibility matrix with sibling libraries (`smonitor`, `depdigest`, `pyunitwizard`).
- Performance and reliability hardening:
  - deterministic loader order,
  - cache and registry read/write safety,
  - import and overhead regression checks.
- Ecosystem health command:
  - `argdigest health-check`.

## RC Consolidation Policy

During `0.9.x`:

- no new features are accepted,
- only bug fixes, docs polish, and release-gate fixes are accepted,
- regressions found in real usage are fixed in `0.9.x` patches.

## Promotion Criteria to 1.0.0

- remote workflows green for CI/docs/conda on `0.9.0` and any `0.9.x` patches,
- no blocker issues open at the end of the consolidation window,
- `devguide/1.0.0_checklist.md` fully green.
