# 0.8.x Release Candidate Checklist

This checklist operationalizes the `0.8.x` stage in `devguide/ROADMAP.md`.
Its goal is to validate release-candidate readiness before `1.0.0`.

## Status Legend

- `[x]` complete and verified
- `[ ]` pending
- `[-]` intentionally deferred

## A) Packaging and distribution

- [x] validate wheel build/install on CI and local clean env.
- [ ] validate conda recipe build for supported Python versions.
- [x] verify package metadata, dependencies, and entry points.

## B) Quality gates

- [ ] CI test workflow green on target branch.
- [ ] docs workflow green and gh-pages updated.
- [ ] coverage remains at agreed baseline or higher.
- [ ] lint gate green.

## C) Documentation final pass

- [ ] user docs aligned with frozen API.
- [ ] developer docs aligned with release process.
- [ ] API reference generated and consistent with `argdigest.__all__`.

## D) Integration confidence

- [ ] run representative integration checks against MolSysMT target paths.
- [ ] verify diagnostics quality and strictness behavior in migration scenarios.
- [ ] log any remaining non-blocking gaps as post-1.0 items.

## E) RC sign-off

- [ ] no blocker bugs open in core digestion/config/registry paths.
- [ ] release notes draft prepared.
- [ ] approve `0.8.x` RC tag candidate.
