# 0.8.x Release Candidate Checklist

This checklist operationalizes the `0.8.x` stage in `devguide/ROADMAP.md`.
Its goal is to validate release-candidate readiness before `1.0.0`.

## Status Legend

- `[x]` complete and verified
- `[ ]` pending
- `[-]` intentionally deferred

## A) Packaging and distribution

- [x] validate wheel build/install on CI and local clean env.
- [x] validate conda recipe build for supported Python versions.
- [x] verify package metadata, dependencies, and entry points.

## B) Quality gates

- [x] CI test workflow green on target branch.
- [x] docs workflow green and gh-pages updated.
- [x] coverage remains at agreed baseline or higher.
- [x] lint gate green.

## C) Documentation final pass

- [x] user docs aligned with frozen API.
- [x] developer docs aligned with release process.
- [x] API reference generated and consistent with `argdigest.__all__`.

## D) Integration confidence

- [x] run representative integration checks against MolSysMT target paths.
- [x] verify diagnostics quality and strictness behavior in migration scenarios.
- [x] log any remaining non-blocking gaps as post-1.0 items (none identified in current RC checks).

## E) RC sign-off

- [x] no blocker bugs open in core digestion/config/registry paths.
- [x] release notes draft prepared.
- [x] approve `0.8.x` RC tag candidate.
