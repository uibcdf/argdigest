# 0.7.x API Freeze Checklist

This checklist operationalizes the `0.7.x` stage in `devguide/ROADMAP.md`.
Its goal is to freeze ArgDigest public contracts before entering `0.9.x` RC.

## Status Legend

- `[x]` complete and verified
- `[ ]` pending
- `[-]` intentionally deferred

## A) Public API surface freeze

- [ ] lock exported symbols in `argdigest.__all__`.
- [ ] lock public constructor/signature behavior for:
  - `arg_digest`,
  - `arg_digest.map`,
  - `argument_digest`,
  - `register_pipeline`,
  - `get_pipelines`.
- [ ] document compatibility policy for `DigestConfig` fields.
- [ ] verify no unintended public import paths are relied on in docs/examples.

## B) Contract tests

- [ ] add tests that assert public symbols exist and remain importable.
- [ ] add signature snapshot tests for key public callables.
- [ ] add behavior-lock tests for strictness modes and config precedence.
- [ ] add regression tests for wheel install/import behavior.

## C) Deprecation policy

- [ ] identify any legacy aliases/behaviors that should be deprecated before `1.0.0`.
- [ ] define timeline and message policy for deprecations (if any).
- [ ] ensure deprecation diagnostics are catalog-based and actionable.

## D) Documentation freeze alignment

- [ ] ensure all user-facing examples use only frozen API.
- [ ] ensure developer docs reflect the exact public contract.
- [ ] ensure API reference pages match exported symbols.

## E) Release gates for 0.7.x closure

- [ ] `pytest` green in CI.
- [ ] `ruff check .` green in CI.
- [ ] docs build and publish pipeline green.
- [ ] no planned breaking API changes remain before `1.0.0`.

## Exit Decision

- [ ] mark `0.7.x` complete and open `0.9.x` RC checklist.
